# Building Mass Generation – Requirements

---

## 1. Principle Summary

The building mass is the **maximal solid volume** from which subtractive operations can later
be applied. It is generated by stacking per-floor extrusions of a user-supplied polygon. Each
floor is an independent extruded solid so that floors can be individually modified (e.g. via
boolean cuts) without affecting adjacent floors.

The model is deliberately simple: one uniform polygon, one uniform floor height, and a fixed
number of floors. Variability in form comes from later subtractive operations (see
*Subtractive Form Generation – Requirements*), not from this stage.

---

## 2. Inputs

| Parameter | Type | Description |
|---|---|---|
| `polygon_points` | `list[tuple[float, float, float]]` | Closed polygon in the XY plane. Z values are ignored during generation. |
| `floor_height` | `float` | Uniform height of each floor in model units. |
| `num_floors` | `int` | Total number of floors to generate. |

---

## 3. Data Model

### 3.1 `FloorData`

Holds all geometry and metadata for a single floor.

```
FloorData
├── index          : int             – 0-based floor number (0 = ground floor)
├── elevation      : float           – Z of the bottom face in model units
├── floor_height   : float           – height of this floor (stored per-floor to allow variable heights later)
├── solid          : TopoDS_Shape    – extruded solid volume (use for Booleans, area/volume analysis, export)
└── polygon_wire   : TopoDS_Shape    – closed wire at bottom face elevation (use for floor-plan rendering)
```

### 3.2 `BuildingMass`

Top-level container for the complete building volume.

```
BuildingMass
├── polygon_points : list[tuple[float, float, float]]   – original input polygon (Z ignored)
├── floor_height   : float                              – uniform floor height
├── num_floors     : int                                – total number of floors
├── floors         : list[FloorData]                   – ordered list, index 0 = ground floor
└── total_height   : float (derived)                   – floor_height × num_floors
```

`total_height` is computed automatically in `__post_init__` and is not a constructor argument.

---

## 4. Generation Algorithm

`BuildingMass.create(polygon_points, floor_height, num_floors)` builds all floors in a single
pass:

```
For each floor i in range(num_floors):
    elevation = floor_height * i

    if elevation == 0:
        floor_face = create_polygon(polygon_points)          # base face at z=0
    else:
        floor_face = translate_shape(base_face, elevation)   # copy moved up

    solid        = extrude_face(floor_face, floor_height)    # prismatic extrusion upward
    polygon_wire = create_wire(polygon_points, z_offset=elevation)

    floors.append(FloorData(index=i, elevation=elevation,
                            floor_height=floor_height,
                            solid=solid, polygon_wire=polygon_wire))
```

Key properties of the algorithm:

- Each `FloorData.solid` is an independent `TopoDS_Shape` — boolean cuts on one floor do not
  affect others.
- `polygon_wire` records the original floor outline at its elevation. After a boolean cut,
  this field is updated by `extract_bottom_wire()` in the subtraction engine.
- The `polygon_points` stored on `BuildingMass` always reflect the **original** maximal
  footprint, regardless of any later subtractions.

## 5. OCC APIs Used

| Operation | OCC API |
|---|---|
| Create planar face from polygon | `create_polygon(polygon_points)` (project wrapper around `BRepBuilderAPI_MakeFace`) |
| Create wire at elevation | `create_wire(polygon_points, z_offset)` |
| Translate face upward | `translate_shape(face, elevation)` (wrapper around `BRepBuilderAPI_Transform`) |
| Extrude face to solid | `extrude_face(face, height)` (wrapper around `BRepPrimAPI_MakePrism`) |

All OCC wrappers live in `src/floorgeneration.py`.

## 7. Open Questions / Future Extensions

| Topic | Note |
|---|---|
| **Variable floor heights** | `FloorData.floor_height` is stored per-floor to support this, but `BuildingMass.create()` currently enforces a uniform height. A `create_variable()` factory accepting a list of heights would be the extension point. |
| **Non-planar polygons** | Input Z values are currently ignored. Supporting sloped or stepped base planes would require changes to `create_polygon` and the extrusion direction. |
| **Gross floor area** | Per-floor area is computed via `GProp_GProps` + `BRepGProp.SurfaceProperties` on `FloorData.solid`. No convenience method exists yet. |
| **IFC / export** | Each `FloorData.solid` is a self-contained `TopoDS_Shape` ready for boolean operations or direct export. IFC storey mapping would use `FloorData.index` and `FloorData.elevation`. |
